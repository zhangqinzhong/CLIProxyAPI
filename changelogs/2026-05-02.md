# 2026-05-02

## Summary

CLIProxyAPI removed the built-in usage statistics stack and shifted usage reporting to the standalone CPA Usage Keeper project. The core project now keeps lightweight request activity data that external services can read through management APIs.

## Highlights

- Removed built-in usage tracking and logging, including the old logger plugin, management usage handlers, TUI usage tab, and related tests.
- Documented that usage statistics are no longer bundled with CLIProxyAPI and CPAMC starting from v6.10.0.
- Added CPA Usage Keeper as the recommended standalone usage statistics service:
  - https://github.com/Willxup/cpa-usage-keeper
- Added `GET /management/api-key-usage` for external consumers.
- Improved API key usage grouping by including `base_url` with the API key, avoiding mixed statistics across different compatible endpoints.
- Added success and failure counters for auth request usage.
- Added recent request tracking in auth records using 10-minute buckets, retaining about 200 minutes of recent activity.
- Kept the `usage-statistics-enabled` management switch for Redis queue usage statistics compatibility.
- Updated release archive naming in `.goreleaser.yml`.
- Reworked request metadata logging helpers to use generic context utilities instead of Gin-specific context handling.
